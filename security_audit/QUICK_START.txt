   ├─ phase4_exploitability.py       Фаза 4: Анализ эксплуатируемости
   ├─ phase5_chain_analyzer.py       Фаза 5: Анализ цепочек
   └─ report_generator.py            Генератор отчётов

   └─ example_run.py         Пример использования


🚀 БЫСТРЫЙ СТАРТ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. УСТАНОВКА ЗАВИСИМОСТЕЙ
   
   pip install pyyaml


2. НАСТРОЙКА КОНФИГУРАЦИИ (опционально)
   
   Откройте config.yaml и при необходимости измените:
   - base_path: путь к целевому приложению
   - exclude_directories: исключаемые директории
   - dangerous_sinks: опасные функции для анализа


3. ЗАПУСК ПОЛНОГО АУДИТА
   
   cd security_audit
   python3 audit_main.py


4. ПРОСМОТР РЕЗУЛЬТАТОВ
   
   cat results/SECURITY_AUDIT_REPORT.md


📊 РЕЗУЛЬТАТЫ АНАЛИЗА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

После выполнения в директории results/ будут созданы:

├─ entrypoints.json            Все точки входа приложения
├─ dataflows.json              Потоки данных с трансформациями
├─ filtered_flows.json         Потоки с сохранённым контролем
├─ discarded_flows.json        Отброшенные потоки с причинами
├─ vulnerabilities.json        Подтверждённые уязвимости
├─ exploit_chains.json         Цепочки эксплуатации
└─ SECURITY_AUDIT_REPORT.md    📄 ФИНАЛЬНЫЙ ОТЧЁТ НА РУССКОМ

✅ СТРОГАЯ 5-ФАЗНАЯ МЕТОДОЛОГИЯ
   Профессиональный подход без спекуляций

✅ ПОЛНОЕ ПОКРЫТИЕ ТОЧЕК ВХОДА
   • HTTP/HTTPS endpoints (все методы)
   • Server-to-server callbacks (webhooks, IPN)
   • API endpoints (REST-like, JSON)
   • AJAX handlers
   • Admin endpoints (включая вне /admin)
   • Installer/Updater endpoints
   • Cron/Worker endpoints

✅ АНАЛИЗ УЯЗВИМОСТЕЙ
   • SQL Injection
   • Command Injection
   • Cross-Site Scripting (XSS)
   • Path Traversal / File Inclusion
   • Code Execution
   • Insecure File Upload
   • Server-Side Request Forgery (SSRF)
   • Arbitrary File Write
   • Authentication/Authorization Bypass

✅ ЦЕПОЧКИ ЭКСПЛУАТАЦИИ
   • File Write → RCE
   • SQL Injection → Auth Bypass
   • XSS → Session Hijacking
   • File Upload → RCE

✅ ДОКАЗАТЕЛЬНАЯ БАЗА
   Для каждой уязвимости:
   - Точное условие эксплуатации
   - Наблюдаемое воздействие
   - Конкретные доказательства
   - Реальное значение

✅ ОТЧЁТНОСТЬ НА РУССКОМ
   Полный отчёт в формате Markdown

   python3 phase4_exploitability.py --filtered results/filtered_flows.json
   python3 phase5_chain_analyzer.py --vulns results/vulnerabilities.json
